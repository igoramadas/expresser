EXPRESSER METRICS

Gather application metrics and generate JSON output to be used by monitoring systems.

Instance Method Details

# (void) init()

Init metrics and set up cleanup timer.

# (Object) start(id, data)

Starts the counter for a specific metric. The data is optional.

Parameters:

  • id ( String ) — ID of the metric to be started.
  • data ( Object ) — Additional info about the current metric (URL data, for example).

Returns:

  • ( Object ) — Returns the metric object to be used later on end.

# (void) end(obj, error)

Ends the counter for the specified metric, with an optional error to be passed along.

Parameters:

  • obj ( Object ) — The metric object started previsouly on start.
  • error ( Object ) — Optional error that ocurred while processing the metric.

# (void) get(id)

Get collected data for the specified metric.

Parameters:

  • id ( String ) — ID of the metric.

# (void) cleanup()

Clean collected metrics by removing data older than X minutes (defined on settings). Please note that this runs on s schedule so you shouldn't need to call it manually, in most cases.

# (Object) output(options)

Generate the JSON output with all metrics.

Parameters:

  • options ( Object ) — Options to filter the output. Available options are same as settings.metrics.

Returns:

  • ( Object ) — JSON output with relevant metrics.
