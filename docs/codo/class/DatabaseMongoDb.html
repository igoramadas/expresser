Overview

EXPRESSER DATABASE - MONGODB

Handles MongoDB database transactions using the native mongodb module. This plugin attaches itself to the main database module of Expresser.

See also:

  • settings.database.mongodb

Instance Method Details

# (Object) init()

Init the MongoDB database module.

Returns:

  • ( Object ) — Returns the MongoDB connection created (only if default settings are set).

# (void) getConnection(connString, options)

Get the DB connection object.

Parameters:

  • connString ( Object ) — The connection string, for example user:password@hostname/dbname.
  • options ( Object ) — Additional options to be passed when creating the DB connection object.

# (void) get(collection, filter, options, callback)

Get data from the database. A collection and callback must be specified. The filter is optional. Please note that if filter has an _id or id field, or if it's a plain string or number, it will be used to return documents by ID. Otherwise it's used as keys-values object for filtering.

Parameters:

  • collection ( String ) — The collection name.
  • filter ( Object ) — Optional, if a string or number, assume it's the document ID. Otherwise assume keys-values filter.
  • options ( Object ) — Options to be passed to the query.
  • callback ( Method ) — Callback (err, result) when operation has finished.

Options Hash: (options):

  • limit ( Integer ) — Limits the resultset to X documents.

# (void) insert(collection, obj, callback)

Add new documents to the database. The options parameter is optional.

Parameters:

  • collection ( String ) — The collection name.
  • obj ( Object ) — Document or array of documents to be added.
  • callback ( Method ) — Callback (err, result) when operation has finished.

# (void) update(collection, obj, options, callback)

Update existing documents on the database. The options parameter is optional.

Parameters:

  • collection ( String ) — The collection name.
  • obj ( Object ) — Document or data to be updated.
  • options ( Object ) — Optional, options to control and filter the insert behaviour.
  • callback ( Method ) — Callback (err, result) when operation has finished.

Options Hash: (options):

  • filter ( Object ) — Defines the query filter. If not specified, will try using the ID of the passed object.
  • patch ( Boolean ) — Default is false, if true replace only the specific properties of documents instead of the whole data, using $set.
  • upsert ( Boolean ) — Default is false, if true it will create documents if none was found.

# (void) remove(collection, filter, callback)

Delete an object from the database. The obj argument can be either the document itself, or its integer/string ID.

Parameters:

  • collection ( String ) — The collection name.
  • filter ( Object ) — If a string or number, assume it's the document ID. Otherwise assume the document itself.
  • callback ( Method ) — Callback (err, result) when operation has finished.

# (void) count(collection, filter, callback)

Count documents from the database. A collection must be specified. If no filter is not passed then count all documents.

Parameters:

  • collection ( String ) — The collection name.
  • filter ( Object ) — Optional, keys-values filter of documents to be counted.
  • callback ( Method ) — Callback (err, result) when operation has finished.
