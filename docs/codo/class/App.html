Overview

EXPRESSER APP

The Express server. By default it will run on HTTP port 8080

See also:

  • settings.app

Property Summary

(Object) server
Exposes the Express HTTP or HTTPS server object.
(Array<Object>) prependMiddlewares
Array of additional middlewares to be used by the Express server. These will be called before anything is processed, so should be used for things that need immediate processing.
(Array<Object>) appendMiddlewares
Array of additional middlewares to be used by the Express server. Please note that if you're adding middlewares manually you must do it BEFORE calling init.

# (void) init()

Init the Express server.

Parameters:

  • options ( Object ) — App init options. If passed as an array, assume it's the array with extra middlewares.

Options Hash: (options):

  • appendMiddlewares ( Array ) — Array with extra middlewares to be loaded.

# (void) configureServer()

Configure the server. Set views, options, use Express modules, etc.

# (void) startServer()

Start the server using HTTP or HTTPS, depending on the settings.

# (void) kill()

Kill the underlying Express server and shut down the app.

# (void) renderView(req, res, view, options)

Helper to render Pug views. The request, response and view are mandatory, and the options argument is optional.

Parameters:

  • req ( Object ) — The request object.
  • res ( Object ) — The response object.
  • view ( String ) — The Pug filename.
  • options ( Object ) — Options passed to the view, optional.

# (void) renderJson(req, res, data)

Render response as human readable JSON data.

Parameters:

  • req ( Object ) — The request object.
  • res ( Object ) — The response object.
  • data ( Object ) — The JSON data to be sent.

# (void) renderImage(req, res, filename, options)

Render response as image.

Parameters:

  • req ( Object ) — The request object.
  • res ( Object ) — The response object.
  • filename ( String ) — The full path to the image file.
  • options ( Object ) — Options passed to the image renderer, for example the "mimetype".

# (void) renderError(req, res, error, status)

Send error response as JSON. When the server can't return a valid result, send an error response with the specified status code and error output.

Parameters:

  • req ( Object ) — The request object.
  • res ( Object ) — The response object.
  • error ( Object ) — The error object or message to be sent to the client.
  • status ( Integer ) — The response status code, optional, default is 500.
