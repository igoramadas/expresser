Overview

EXPRESSER CRON

Handle scheduled cron jobs. You can use intervals (seconds) or specific times to trigger jobs, and the module will take care of setting the proper timers. Jobs are added using "job" objects with id, schedule, callback and other options.

Examples:

Sample job object, alerts user every minute (60 seconds). 

var myJob = {
  id: "alertJob",
  schedule: 60,
  callback: function(job) { alertUser(mydata); }
}

Sample job object, sends email every day at 10AM and 5PM. 

var otherJob = {
  id: "my mail job",
  schedule: ["10:00:00", "17:00:00"],
  callback: function(job) { mail.send(something); }
}

Instance Method Details

# (void) init()

Init the cron manager. If loadOnInit setting is true, the cron.json file will be parsed and loaded straight away (if there's one).

# (void) setEvents()

Bind events.

# (void) load(filename, options)

Load jobs from the specified (default cron.json) file. If autoStart is true, it will automatically call the start method after loading.

Parameters:

  • filename ( String ) — Path to the JSON file containing jobs, optional, default is "cron.json".
  • options ( Object ) — Options to be passed when loading cron jobs.

Options Hash: (options):

  • filename ( String ) — Name of file to be loaded (in case filename was not set on first parameter)
  • basePath ( String ) — Sets the base path of modules when requiring them.
  • autoStart ( Boolean ) — If true, call start after loading.

# (void) start(idOrFilter)

Start the specified cron job. If no id is specified, all jobs will be started. A filter can also be passed as an object. For example to start all jobs for the module "email", use start({module: "email"}).

Parameters:

  • idOrFilter ( String ) — The job id or filter, optional (if not specified, start everything).

# (void) stop(idOrFilter)

Stop the specified cron job. If no id is specified, all jobs will be stopped. A filter can also be passed as an object. For example to stop all jobs for the module "mymodule", use stop({module: "mymodule"}).

Parameters:

  • idOrFilter ( String ) — The job id or filter, optional (if not specified, stop everything).

# (Object) add(job)

Add a scheduled job to the cron, passing a job. You can also pass only the job if it has an id property.

Parameters:

  • id ( String ) — The job ID, optional, overrides job.id in case it has one.
  • job ( Object ) — The job object.

Options Hash: (job):

  • id ( String ) — The job ID, optional.
  • schedule ( Integer, Array ) — If a number assume it's the interval in seconds, otherwise a times array.
  • callback ( Method ) — The callback (job) to be triggered.
  • once ( Boolean ) — If true, the job will be triggered only once no matter which schedule it has.

Returns:

  • ( Object ) — Returns {error, job}, where job is the job object and error is the error message (if any).

# (void) remove(id)

Remove and stop a current job. If job does not exist, a warning will be logged.

Parameters:

  • id ( String ) — The job ID.
