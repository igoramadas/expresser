Overview

EXPRESSER SOCKETS

Handles sockets communication using the module Socket.IO. ATTENTION! The Sockets module is started automatically by the App module. If you wish to disable it, set Settings.sockets.enabled to false.

See also:

  • settings.sockets

Instance Method Details

# (void) init()

Init the Sockets plugin.

# (void) setEvents()

Bind events.

# (void) bind(server)

Bind the Socket.IO object to the Express app. This will also set the counter to increase / decrease when users connects or disconnects from the app.

Parameters:

  • options ( Object ) — Sockets init options.

Options Hash: (options):

  • server ( Object ) — The Express server object to bind to.

# (void) emit(key, data)

Emit the specified key and data to clients.

Parameters:

  • key ( String ) — The event key.
  • data ( Object ) — The JSON data to be sent out to clients.

# (void) listenTo(key, callback, onlyNewClients)

Listen to a specific event. If onlyNewClients is true then it won't listen to that particular event from currently connected clients.

Parameters:

  • key ( String ) — The event key.
  • callback ( Method ) — The callback to be called when key is triggered.
  • onlyNewClients ( Boolean ) — Optional, if true, listen to event only from new clients.

# (void) stopListening(key, callback)

Stops listening to the specified event key.

Parameters:

  • key ( String ) — The event key.
  • callback ( Object ) — The callback to stop triggering.

# (void) compact()

Remove invalid and expired event listeners.

# (void) onDisconnect()
